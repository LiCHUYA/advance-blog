1.作用域提升的定义 ​

作用域提升是指在JavaScript中，变量声明和函数声明在编译阶段被提升到它们所在作用域的顶部执行，而不是在代码中实际出现的位置。这意味着在代码执行之前，变量和函数已经在作用域内声明好了，但变量的赋值操作会保持在原地。

2. 作用域提升的类型 ​

作用域提升分为两种：变量提升和函数提升。

2.1 函数提升 ​

在提升过程中，函数声明会被提升到当前作用域的顶部。整个函数的定义，包括其函数体，在代码运行前已经在内存中可用。

示例：

javascript
foo(); // 输出: "Hello"

function foo() {
  console.log("Hello");
}

在这段代码中，虽然foo函数是在调用它之后定义的，但由于函数提升，JavaScript引擎会在执行代码时将foo函数提升到作用域的顶部，因此foo()能够正常调用并输出“Hello”。

2.2 变量提升 ​

变量声明（使用var）会被提升到当前作用域的顶部，但仅提升声明部分，而不是赋值部分。因此，在变量声明前访问该变量时，会返回undefined。

示例：

javascript
console.log(a); // 输出: undefined
var a = 5;
console.log(a); // 输出: 5

在这段代码中，var a的声明被提升到顶部，但赋值操作a = 5依然在原位置。因此，在第一次console.log(a)时，a已经被声明，但还未赋值，所以输出undefined。在第二次console.log(a)时，a已经被赋值为5，因此输出5。

3. 函数与变量的提升优先级 ​

当一个作用域内存在同名的变量和函数时，函数提升的优先级高于变量提升。

示例：

javascript
console.log(a); // 输出: function a() {}
var a = 1;
function a() {}
console.log(a); // 输出: 1
  1. 先进行函数提升，function a() {}被提升到顶部。
  2. 然后是变量声明的提升，var a;被提升到顶部，但此时被函数声明覆盖。
  3. 执行console.log(a)时，a是函数，因此输出函数体。
  4. 之后，a = 1;的赋值操作将a的值从函数覆盖为数字1。
  5. 再次console.log(a)时，输出1。

4. let和const的作用域提升 ​

与var不同，let和const声明的变量也会被提升，但在提升过程中，这些变量会处于暂时性死区（Temporal Dead Zone, TDZ），即在变量声明前访问它们会抛出ReferenceError。

示例：

javascript
console.log(b); // ReferenceError: Cannot access 'b' before initialization
let b = 3;

解释：

在这段代码中，let b的声明被提升了，但在声明之前访问b会导致ReferenceError，因为它处于暂时性死区（TDZ）。

5. 函数表达式与箭头函数的提升 ​

函数表达式和箭头函数在提升时只会提升变量声明，而不会提升函数本身。这意味着在声明之前调用它们会导致TypeError或undefined。

示例：

javascript
console.log(foo); // 输出: undefined
foo(); // TypeError: foo is not a function

var foo = function() {
  console.log("Hello");
};

解释：

在这段代码中，var foo的声明被提升，但foo本身是一个函数表达式，没有提升，因此第一次console.log(foo)输出undefined，第二次foo()调用会抛出TypeError。

javascript

    var a = 2;
    b = c = 20;
    function inner() {
      console.log(a);
    }

    inner();
  }

  var a = 3;
  middle();
  console.log(b);
}

题目 2：闭包中的提升 ​

javascript
function counter() {
  console.log(count);
  var count = 1;

  function increment() {
    count++;
    console.log(count);
  }

  return increment;
}

var inc = counter();
inc();
inc();

问题： 运行这段代码后，控制台的输出是什么？解释闭包与变量提升是如何影响结果的。

题目 3：作用域链中的变量与函数提升 ​

javascript
var b = 10;

function test() {
  console.log(b); 

  if (typeof b === 'function') {
    b();
  }

  function b() {
    console.log("Function b called");
  }
}

test();
console.log(b);

问题： 上述代码会输出什么？详细解释作用域链、函数提升和typeof的交互是如何影响执行结果的。

javascript
function foo(x) {

  } else {

javascript

console.log(mystery(6));

javascript

    var a = 2;

foo();

题目9：多种函数类型与变量提升 ​

js
var func = "Global variable";

function func() {
  console.log("Global function");
}

// 立即执行函数 (IIFE)
(function() {
  console.log(func); // 第一次输出

  var func = "IIFE variable";

  // 箭头函数
  const funcArrow = () => {
    console.log(func); // 第二次输出
  };

  funcArrow();

  // 普通函数
  function func() {
    console.log("IIFE function");
  }

  console.log(func); // 第三次输出

  // 构造函数
  function FuncConstructor() {
    this.func = "Constructor property";
    console.log(this.func); // 第四次输出
  }

  new FuncConstructor();

  console.log(func); // 第五次输出
})();

console.log(func); // 第六次输出
