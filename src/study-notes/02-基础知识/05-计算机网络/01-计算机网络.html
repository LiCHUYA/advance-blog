计算机网络 ​

1.协议 ​

计算机网络协议是指计算机网络中用于通信的规则和约定的集合。这些协议定义了如何在网络中传输数据，确保不同设备之间能够顺利地进行通信。网络协议可以分为多个层次，每个层次都有不同的功能和特点。

主要的计算机网络协议 ​

  1. 物理层协议

    • Ethernet（以太网）：是一种常见的局域网技术，定义了网络设备如何在物理介质上传输数据，主要使用双绞线、光纤等介质。
    • DSL、光纤协议：用于宽带接入网络，提供高速数据传输。

  2. 数据链路层协议

    • PPP（点对点协议）：用于直接连接两个网络节点，常用于拨号和宽带连接。
    • MAC（介质访问控制）协议：管理网络设备如何共享通信介质，常见的如 CSMA/CD（载波监听多路访问/冲突检测）在以太网中使用。
    • ARP（地址解析协议）：用于将 IP 地址映射为物理地址（MAC 地址），确保数据包能够正确地传送到目的设备。

  3. 网络层协议

    • IP（互联网协议）：负责数据包的寻址和路由，是整个互联网的核心协议。IP 分为 IPv4 和 IPv6，IPv6 提供了更大的地址空间。
    • ICMP（互联网控制消息协议）：用于发送错误报告和其他控制消息，例如 ping 命令使用的就是 ICMP。
    • OSPF（开放最短路径优先协议）：一种路由协议，用于在网络中动态更新路由信息。

  4. 传输层协议

    • TCP（传输控制协议）：提供可靠的、面向连接的数据传输，确保数据按顺序到达并且无误。TCP 适用于需要可靠传输的应用，如网页浏览、文件传输等。
    • UDP（用户数据报协议）：提供不可靠的、无连接的数据传输，适用于实时应用，如视频流、在线游戏等，不要求数据必须按顺序到达。

  5. 会话层协议

    • RPC（远程过程调用）：允许程序在网络上调用另一个程序中的方法或过程。
    • NetBIOS：提供在局域网中进行会话建立、名称解析等服务。

  6. 表示层协议

    • SSL/TLS（安全套接字层/传输层安全协议）：用于加密传输层数据，确保数据在传输过程中不会被窃取或篡改。HTTPS 就是基于 SSL/TLS 的 HTTP 协议。
    • MIME（多用途互联网邮件扩展）：用于表示电子邮件中的多媒体内容，使得电子邮件能够包含文本、图像、音频等多种格式。

  7. 应用层协议

    • HTTP/HTTPS（超文本传输协议/安全超文本传输协议）：用于万维网中的数据传输，HTTP 是无状态的协议，HTTPS 则在 HTTP 基础上加入了 SSL/TLS 加密。
    • FTP（文件传输协议）：用于在计算机之间传输文件，支持文件的上传和下载。
    • SMTP（简单邮件传输协议）：用于电子邮件的发送。
    • POP3/IMAP（邮局协议版本3/互联网消息访问协议）：用于电子邮件的接收，POP3 常用于下载邮件到本地，IMAP 则可以在服务器上管理邮件。
    • DNS（域名系统）：用于将域名解析为 IP 地址，便于人类记忆和访问网络资源。
    • DHCP（动态主机配置协议）：用于自动分配 IP 地址及其他网络配置，减少手动配置的工作量。

网络协议分层模型 ​

网络协议通常被组织为分层模型，其中最著名的是 OSI 七层模型 和 TCP/IP 四层模型。

  • OSI 七层模型：从下到上依次是物理层、数据链路层、网络层、传输层、会话层、表示层和应用层。
  • TCP/IP 四层模型：从下到上依次是网络接口层、互联网层、传输层和应用层。这个模型更贴近实际使用，特别是在互联网中。
